01 — The Pipeline

Vision to Execution

Everything starts as a thought. The pipeline narrows intent into executable work, one level at a time — from the abstract to the concrete.

Vision

The North Star

What am I building and why? A single document that captures the overarching purpose. Everything downstream exists to serve this. It rarely changes, but everything references it.

docs/vision.md

Goals

Measurable Outcomes

Concrete targets with success criteria. Each goal has a horizon (short/medium/long), a motivation section, and measurable success criteria. Goals keep ideas honest.

horizon: short | medium | long § Success Criteria § Motivation

Ideas

Exploration Space

Raw sparks that evolve through research. Ideas are aligned to goals and progress through stages: spark → exploring → ready → promoted. A promoted idea becomes a project.

spark exploring ready parked promoted

Projects

Multi-Plan Efforts

Orchestrate sequential (or parallel) plans. A project is a home document with an execution section — an ordered list of plan links. The project builder walks through them one by one, dispatching each to an agent.

status: draft → approved → building → completed § Execution Order ← hasMany Plans

Plans

Agent-Executable

The atomic unit of AI work. A plan is a Claude Code /plan-style document with references, a test plan, and approval status. Approved plans get picked up and executed. Cost, turns, and tool calls are recorded on completion.

status: pending → approved → completed § References § Test Plan costUsd · turns · toolCalls

03 — The Executors

Plays & Tasks

Two kinds of executable prompts drive the Act phase. Plays repeat on a schedule — they're the system's habits. Tasks fire once — they're the system's errands.

Play · Repeatable

The System's Habits

Plays are markdown prompts that run on a schedule. They're the things the system does every day, every hour, at the beginning of the day, at the end. Like habits, they compound.

build-an-approved-project Pick up the next approved project and execute its plans every-half-hour
begin-exploring-one-of-the-spark-ideas Find a spark idea, research it, evolve to exploring beginning-of-day
Task · One-off

The System's Errands

Tasks are markdown prompts that run once and get marked complete. Small fixes, investigations, one-off reports, documentation updates. They clear the backlog.

investigate-session-memory Debug why session memory isn't persisting one-off
import-gws-cli-recipes Download and organize Google Workspace docs one-off

# Build an Approved Project

Query contentbase for projects with status "approved".
Pick the oldest one. Find its next unbuilt plan.
Execute the plan. Record cost and completion status.
If all plans are done, mark the project completed.

## Conditions

```ts
// Only run if there's an approved project waiting
const projects = await docs.query(docs.models.Project)
  .where("meta.status", "approved").fetchAll()
return projects.length > 0
```

06 — The Tempo

Speed of the Loop

Boyd's key insight: the side that cycles faster wins. Not faster code — faster learning. The agentic loop compresses the time between observation and action.

V

Vision

Written once. Referenced always. The constant against which everything is measured.

→
G

Goals

Updated weekly. Success criteria get checked by end-of-day plays and reports.

→
I

Ideas

Explored daily. Beginning-of-day plays evolve sparks. You promote the winners.

→
Pj

Projects

Approved and queued. The project builder picks them up every 30 minutes.

→
Pl

Plans

Executed in minutes. Cost and metrics recorded. Results feed the next Observe.

Write a goal on Monday. An idea gets explored Tuesday morning. It's promoted to a project by lunch. Plans are approved by evening. By Wednesday, running software. By Thursday, a report tells you what to do next. — The tempo of the agentic loop

// The feedback instruments
