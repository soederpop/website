01 — Foundation

Content Models

Everything starts as structured markdown. Each document type has a schema, sections with extractors, relationships to other models, and computed properties. This is your single source of truth.

02 — Entry Points

How Intent Enters the System

Tasks and ideas can originate from anywhere — your phone, your voice, your editor, a schedule, or a Telegram message. Each channel feeds into the same content model layer.

On the go

Contentbase Mobile App

Create tasks, ideas, and plans from your phone. The app talks to the contentbase HTTP API. Write a quick task in the field, and it's waiting when the agent picks it up.

At desk

Editor + Claude Code

Write structured markdown documents directly. Claude Code sessions can create plans, projects, and tasks through the MCP server or by editing files in the docs/ folder.

Voice

luca voice + Launcher App

A Spotlight-style native macOS app transcribes short voice commands and dispatches them over IPC to luca voice, which routes them to registered handlers.

Remote

Telegram Bot

The grammY-powered Telegram feature can receive commands from anywhere. Register custom handlers that create tasks or trigger actions.

Scheduled

Heartbeat Scheduler

The heartbeat feature ticks on configurable intervals — every 30 minutes, hourly, three times daily, end of day. Each tier can execute code plays or dispatch prompt plays to Claude Code or Codex.

Auto

luca main — Task Scanner

Runs continuously. Reads the docs/tasks/ collection, matches documents by their schedule meta field, and dispatches them to the configured agent at the right time.

04 — Executable Intent

Tasks as Agents

A task is a markdown document with frontmatter. The schedule determines when it runs. The body is the prompt. The agent field determines who executes it.

↻ Repeatable · beginning-of-day

develop-my-ideas.md

Evaluate current goals, review recent work context, and promote the most promising idea into a full project with sequenced plans ready for approval.

⚡ claudeCode
↻ Repeatable · every-half-hour

build-the-next-approved-project

Query contentbase for the oldest approved project. Pick up the next unbuilt plan, execute it in a Claude Code session, record cost and completion status.

⚡ claudeCode
◆ One-shot

⚡ claudeCode
◆ One-shot

⚡ claudeCode
↻ Repeatable · end-of-day

daily-summary.md

Potential: Summarize what was built, committed, deployed. Update goals progress. Generate a report in docs/reports with the day's outcomes.

⚡ luca assistant
↻ Repeatable · hourly

06 — Always Running

The Persistent Daemons

Three long-running processes form the backbone. They start when your machine boots and provide continuous orchestration, voice control, and web serving.

luca main

Portfolio Orchestrator

Reads Domain documents. Spawns service processes for each subdomain. Health-checks every 60 seconds. Auto-restarts failures. + Task Scanner: polls docs/tasks for scheduled work, dispatches to the right agent at the right time.

luca voice

Voice Command Router

Listens on an IPC socket for transcribed voice commands from the native macOS launcher. Routes commands to registered handlers. Supports ElevenLabs and Chatterbox TTS for responses.

cloudflared

Tunnel to the World

Cloudflare tunnel exposes local services to the internet with SSL. soederpop.com subdomains resolve through the tunnel to local ports managed by luca main.
